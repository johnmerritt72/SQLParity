# What's New in SQLParity 1.3.6

This release focuses on making the side-by-side DDL comparison panels easier to read and to work with.

## ✨ Highlights

### 🔍 Zoom the DDL panels
Resize the comparison text on the fly to suit your monitor and eyesight:

- **Ctrl + Mouse Wheel** — scroll up/down to grow/shrink the font
- **Ctrl + Plus / Ctrl + Minus** — step the size up or down
- **Ctrl + 0** — reset to the default size

Your chosen size sticks between sessions, and you can also set it under **Tools → Options → SQLParity → "DDL Panel Font Size"**.

### ↩️ Word-wrap toggle
A new **"Wrap"** checkbox sits in the panel header. Leave it off (the default) to keep each statement on one line and scroll horizontally — handy for reading SQL structure — or turn it on to wrap long lines to the panel width. The setting applies to both panels and is remembered between sessions (**Tools → Options → SQLParity → "Wrap DDL Panels"**).

### 📋 Copy clean SQL — no more line numbers
Line numbers now live in a dedicated gutter alongside the SQL instead of being mixed into the text. That means when you select and copy DDL from a panel, you get **clean, runnable SQL** — no line-number prefixes to strip out before executing. Line numbers stay perfectly aligned whether word-wrap is on or off, and you can still hide them entirely via **Tools → Options → SQLParity → "Show Line Numbers"**.

## 🐛 Also included
- **Side label now refreshes after "Start Over."** Picking a different server from the Side A / Side B dropdown after Start Over no longer leaves the label showing the previous server, and no longer risks corrupting saved labels in your connection history. (Originally fixed in 1.3.5.)
